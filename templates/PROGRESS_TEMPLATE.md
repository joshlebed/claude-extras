## Instructions for Planning Agent

When creating this PROGRESS.md for a new project:

### 1. Fill In Header

- Update `[PROJECT NAME]` with actual name
- Set initial task count `(0/X tasks)`
- Estimate total remaining time

### 2. Create Task Breakdown

For each task include:
- **Checkbox** `- [ ]` for tracking
- **Bold task name** that's specific and actionable
- **File path** - exact file to modify
- **Action** - one-sentence description
- **Estimate** - realistic time (5 min, 15 min, 30 min, 1 hour)

### 3. Prioritize Correctly

**High Priority:**
- Core functionality that blocks other work
- Security or data integrity issues
- Features users will immediately notice

**Medium Priority:**
- Edge cases for core features
- Developer experience improvements
- Non-critical UI polish

**Low Priority:**
- Optimizations
- Cleanup and refactoring
- Documentation updates

### 4. Set Next Priority

Always have 3 clear "next up" tasks so the implementing agent knows where to start.

### 5. Time Estimation Guide

- Simple change (add import, modify prop): 5-10 min
- Component update (add hook, modify logic): 15-30 min
- New component or complex refactor: 30-60 min
- Add 50% buffer for unknowns

### 6. Delete This Section

Remove "Instructions for Planning Agent" before saving.
